## 四、核心痛点与矛盾

### 4.1 痛点一：集约化 vs 灵活性的结构矛盾 ⚠️ 高影响

```
集约化 Infra + Serving
  ✅ 统一芯片选型 → 规模效应降本
  ✅ 框架/算子标准化 → 减少重复建设
  ❌ 业务形态太杂 → 集中Serving Oncall爆炸
  ❌ 新业务接入周期长 → 灵活性丧失

具体表现：
  - 豆包(500ms首Token) vs 视频生成(分钟级) → 完全不同的调度策略
  - Trae(流式长上下文) vs Agent(7×24持久) → 不同内存管理模式
  - 推荐/广告(<5ms) vs MaaS(弹性) → 不同SLA要求
```

**→ TA切入点**：芯片分级部署方案（不同场景→不同芯片池→不同SLA等级），统一调度层+差异化底层 → 既要集约化降本，又保持灵活性。

### 4.2 痛点二：GPU→NPU迁移阵痛 ⚠️ 高影响

| 痛点 | 表现 | 根因 |
|------|------|------|
| 框架适配拖慢节奏 | VeOmni/veRL需同时维护GPU+NPU代码路径 | CUDA/CANN生态差异 |
| 训练精度不对齐 | GPU vs NPU训练收敛不一致 | 混合精度策略差异(BF16/FP16)，CANN升级引入回归 |
| 推理精度不对齐 | 同模型GPU vs NPU推理输出分布不同 | 算子实现差异累积 |
| 每次CANN升级 → 回归 | 适配周期长 | 缺少自动化精度/性能回归测试工具链 |

> 🟡 这是Gary的行业洞察，有多家国产芯片企业的公开发声佐证（如昇腾适配案例）。

**→ TA切入点**：精度对齐自动化工具链 + 迁移流程标准化 + 跨芯片CI/CD。不是卖芯片，是卖"迁移能力"。

### 4.3 痛点三：推理成本与MaaS盈利的压力 ⚠️ 高影响

```
推理规模爆炸：
  日均120万亿Token × 3个月翻倍趋势
  × MaaS收入爆发（Q1超全年目标10倍）
  × 但MaaS天然低毛利（Token单价持续压缩）

成本算术：
  每个百分点的推理成本降低 ≈ 数十亿人民币/年
  → 推理TCO优化不是技术问题，是商业模式的基础
```

**→ TA切入点**：推理TCO全栈优化 — 不只在芯片层，还包括Serving引擎优化（KV Cache量化/Continuous Batching/Prefix Caching）、模型路由（简单问题→小模型）、芯片分级。

### 4.4 痛点四：芯片供应多线作战 ⚠️ 中高影响

```
必须同时维护：
  英伟达存量（断供但仍在用）→ 维持现状
  昇腾910C/950PR → 训练+推理主力
  寒武纪590 → 推理扩容
  高通AI ASIC → 推理新选项（传闻数百万颗订单）
  自研Ada（三星6nm）→ 长期自主可控

→ 五套工具链 + 五套部署方案 + 五套运维体系
→ 这是Infra层的噩梦
```

**→ TA切入点**：多芯片统一管理平台 → 一套调度层管理所有芯片，向上对业务透明。

### 6.2 字节的优势与软肋

| 优势 | 软肋 |
|------|------|
| MaaS调用量第一 → 规模效应 | 英伟达依赖度高 → 断供影响最大 |
| 自有场景（抖音/豆包/即梦）→ 芯片需求量大且多元 | 芯片迁移包袱最重 → GPU→NPU全栈迁移 |
| 自有现金流转正 → 有力支撑Capex | 组织架构调整频繁 → 决策链可能变化 |
| 自有推理引擎和训练框架(VeOmni/veRL) → 技术自主 | 国产芯片适配起步晚于华为/阿里 |

**→ TA含义**：字节是国产芯片供应商的"最佳客户"——需求最大、场景最全、付费能力最强。但也因此对方案完整度要求最高：不是能跑就行，是要能跑得比GPU好（或接近）才行。
